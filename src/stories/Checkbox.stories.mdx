# Checkbox

O componente `Checkbox` deve ser utilizado em casos no qual o usuário pode selecionar mais de uma opção dentre uma lista de opções.
Em sua versão desktop os Checkboxes são mostrados lado a lado. No mobile, os Checkboxes são mostrados verticalmente.
É composto por subcomponentes e propriedades que podem ou não ser acrescentados à sua estrutura principal, de acordo com a necessidade e preferência do usuário.
Sendo eles:

- `name`: Define o nome do `input`. Obrigatória para sabermos quais botões pertencem a este `name`.
- `label`: Texto do `input`, recebe uma string que fica posicionada ao lado do ícone.
- `value`: Define qual o valor do componente, pode receber uma string, um numeral ou até um booleano.
- `disabled`: Define se o componente está ou não habilitado, recebe um booleano.
- `defaultValue`: Define o valor inicial do input.
- `checked`: Define se o componente está marcado ou se não, recebe um booleano.
- `color`: Define a cor do texto referente àquele `Checkbox`, recebe uma string informando a cor.
- `onChange`: Define o que será executado quando o componente for modificado.
- `id`: Define o `id` do `input`.

O `Checkbox` pode receber a prop `checked`. É um valor booleano que indica o valor se o valor está selecionado ou não.

# Color | ColorIcon

O `Checkbox` pode receber as seguintes propriedades para cores: `color` e `colorIcon`.
A propriedade `colorIcon` indica qual do ícone apenas, enquanto a propriedade `color` indica a cor do ícone e do texto.
Se o componente estiver desabilitado, a cor do ícone e texto será `gray.500`.
